Project Documentation: Referral & Wallet App

Version 1.1 | August 27, 2025

1. Project Overview & Goals

This document outlines the requirements for a mobile application designed to manage a customer referral program. The primary goal is to provide a platform for users to earn wallet cash by referring financial products (e.g., credit cards) and to allow an administrator to manage users, referrals, and payout requests through a dedicated web panel.

The system is composed of two main components, each designed for its specific user base to ensure a seamless and efficient experience for both customers and administrators.

Customer-Facing Mobile App

An iOS & Android application for users to manage their profile, access referral links, track earnings, and redeem their wallet balance.

Secure Admin Panel

A web-based dashboard for the client to manage redemption requests, credit user wallets, and oversee all platform activity in real-time.

2. Core Features & Functionality

This section breaks down the core features of both the customer-facing mobile app and the secure admin panel. Use the tabs below to explore the functionality planned for each component.

User Authentication

Secure registration and login system for customers.

Home Dashboard

Displays wallet balance, tier level, and a list of clickable referral offers.

Referral Offer Details

A dedicated screen for each offer showing benefits, features, and full T&Cs.

Wallet & Redemption

Allows users to initiate payouts by entering their UPI ID and desired amount.

Editable User Profile

Users can view and edit their information, such as name and UPI ID.

Transaction History

Provides a clear history of all earnings and redemptions.

Secure Admin Login

A separate, secure login page. Admin accounts are created manually in the backend.

Dashboard Overview

Presents key metrics like total pending requests and total users at a glance.

Redemption Management

A real-time view of all redemption requests with functionality to mark them as "Completed".

User Management

Allows the admin to view all users and manually add funds to their wallets for successful referrals.

5. API Endpoint Definitions

The backend will expose a RESTful API to enable communication between the mobile app, admin panel, and the database. Below are the preliminary definitions for the key endpoints.

Method Endpoint Description Protection
POST/api/auth/registerRegister a new user.Public
POST/api/auth/loginLog in a user or admin.Public
GET/api/user/profileGet the logged-in user's profile.User
PUT/api/user/profileUpdate the logged-in user's profile.User
POST/api/user/redeemSubmit a redemption request.User
GET/api/offersGet all available referral offers.Public
GET/api/admin/requestsGet all redemption requests.Admin
PUT/api/admin/requests/:idUpdate a redemption request's status.Admin
PUT/api/admin/users/:userId/balanceManually add funds to a user's wallet.Admin
